# 🚀 Quick Deployment Reference

## ⚡ Super Quick Start (5 Minutes)

### 1️⃣ Push to GitHub
```bash
git init
git add .
git commit -m "ISL Translation System"
git remote add origin https://github.com/YOUR_USERNAME/isl-translation.git
git push -u origin main
```

### 2️⃣ Deploy Backend (Render)
1. Go to [render.com](https://render.com)
2. New + → Blueprint
3. Connect GitHub repo
4. Select `render.yaml`
5. Click "Apply"
6. **Copy backend URL**: `https://isl-backend-xxxx.onrender.com`

### 3️⃣ Deploy Frontend (Vercel)
1. Go to [vercel.com](https://vercel.com)
2. New Project → Import GitHub repo
3. Settings:
   - Root: `isl-frontend`
   - Framework: `Vite`
   - Env vars:
     ```
     VITE_API_URL=https://isl-backend-xxxx.onrender.com
     VITE_SOCKET_URL=https://isl-backend-xxxx.onrender.com
     ```
4. Deploy!

## 🤖 ML Model Options

### Option A: Small Models (< 100MB)
```bash
# Just commit them
git add isl-backend/ml_models/*.pkl
git commit -m "Add models"
git push
```

### Option B: Large Models (> 100MB)
1. Upload to Google Drive
2. Get shareable link
3. Update `download_models.py`:
   ```python
   models = {
       "gesture_model.pkl": {
           "url": "YOUR_GOOGLE_DRIVE_LINK",
           "required": True
       }
   }
   ```
4. Models download automatically on startup!

## 🐛 Common Issues

| Issue | Solution |
|-------|----------|
| Camera not working | Must use HTTPS (Vercel provides this) |
| Backend sleeping | Free tier sleeps after 15min (wakes in 30sec) |
| Models not loading | Check `download_models.py` URLs |
| Database error | Check `DATABASE_URL` in Render |
| Build failed | Check logs in Render/Vercel dashboard |
